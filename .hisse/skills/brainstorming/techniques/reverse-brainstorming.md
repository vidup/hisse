# Brainstorming Inversé

Technique créative où l'on réfléchit à comment ÉCHOUER plutôt qu'à comment réussir.
En identifiant tous les moyens de rater, on découvre ce qu'il faut éviter — et donc
ce qu'il faut faire.

## Quand l'utiliser

- Anticiper les problèmes de manière proactive
- Quand les solutions directes ne viennent pas
- Stress-tester un concept
- Cartographier les risques d'un projet

## Processus

1. **Formuler** l'objectif clairement
2. **Inverser** la question : "Comment ferait-on pour échouer ?"
3. **Brainstormer** les modes d'échec sans autocensure
4. **Convertir** chaque échec en action positive
5. **Prioriser** par probabilité d'occurrence

## Exemple

**Objectif** : Créer une app de méditation qui sera utilisée.

**Question inversée** : "Comment faire pour que PERSONNE ne l'utilise ?"

| Mode d'échec | Action positive |
|---|---|
| Interface compliquée avec 50 options | Interface minimaliste, 3 clics maximum |
| Inscription obligatoire avant d'essayer | Permettre l'essai immédiat sans compte |
| Notifications agressives toutes les heures | Rappels doux, configurables, opt-in |
| Contenu uniquement en anglais technique | Contenus localisés, langage accessible |
| Nécessiter 30 minutes minimum par session | Sessions de 2 minutes disponibles |

## Conseils

- C'est souvent plus facile de trouver des problèmes que des solutions
- Les modes d'échec spécifiques donnent des stratégies concrètes
- Cette technique fait émerger les hypothèses cachées
- Se combine très bien avec le SWOT ensuite
